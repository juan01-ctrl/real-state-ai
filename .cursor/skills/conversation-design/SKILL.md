# Conversation design

## Purpose

Produce messages a **top broker** could send: **natural**, **high-trust**, **commercially sharp**—so **serious buyers** advance, **follow-up** feels intentional, and **automation** never reads like **bulk** or **generic** “AI.”

## When to use

- WhatsApp / Instagram DM / SMS: **inbound replies**, **outbound**, **nudges**, **post-visit**.
- **Policies**: when to **ask**, **suggest listings**, **offer slots**, **escalate to human**.
- **Hot vs nurture** cadence: spacing, max touches, **backoff** rules.

## When not to use

- **Field schemas and scores** — **lead-qualification** skill (this skill **uses** them as inputs).
- **Ranking listings** math — **property-recommendation** skill.
- **CRM sync rules** — **crm-flow-design** skill.

## Inputs (gather first)

- [ ] **Brand** tone: formal vs warm; **emoji** policy; **language** (locale).
- [ ] **Stage goal** for the block (e.g. confirm **financing mode** + **zone** before visit talk).
- [ ] **Channel limits**: character soft caps, **link** preview behavior, **media** usage.
- [ ] **Quiet hours** and **opt-out** compliance defaults.
- [ ] **Human handoff** triggers (legal, **high value**, frustration, **sensitive**).

## Workflow

1. **Define the single goal** of the next message—**one primary move** (question **or** CTA **or** **two** short listing hooks—not all three).
2. **Anchor in thread** — Reference **specific** prior buyer words; **rotate** openings across sequences.
3. **Progressive trust** — Match broker discovery: **economics** and **constraints** before **calendar** when appropriate.
4. **Objections** — Acknowledge **before** pushing next step (price, timing, “sending options”).
5. **Branching** — If clear answer → next stage; if vague → **one** clarifier; if **red flag** → human.
6. **CRM residue** — After key turns, **one-line note** operators should see (paste-ready).

**Policy addendum**

- **Cadence**: hot (hours), warm (days), nurture (weeks) — **max touches** before pause.
- **Banned phrases** (generic “hope you’re well” spam openers if brand rejects them).
- **Quality bar**: read-aloud test — **credible broker** or **marketing blast**?

## Project-specific considerations

- **Consultative + premium**: **curiosity** and **specificity** beat **hype**; **never** sound like **template** **roulette**.
- **Commercially useful**: every message **earns** a reply or a **clear** next step toward **visit** or **honest disqualify**.
- **Serious buyers**: tighter **questions** for **high-intent**; **lighter** touch for **long timeline** nurture.

## Avoid

- **Robotic** symmetry (“Thank you for your interest”) on every turn.
- **Three asks** in one message.
- **Listing dumps** without **why** they fit (hand off to **property-recommendation** copy patterns).
- **Ignoring** frustration signals — **escalate** early.

## Success metrics

Positive **reply** rate; **time to visit intent**; **human takeover** rate (low except **edge** cases); **complaint** rate.
