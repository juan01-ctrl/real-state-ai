1) Qué es el producto

Aesthete es una capa de inteligencia comercial para inmobiliarias. No está pensado como “otro CRM” ni como “chatbot genérico”: su propuesta central es detectar qué leads realmente importan, recomendar la próxima acción y mostrar dónde se están perdiendo oportunidades.

Superficie actual del producto: /dashboard, /leads, /leads/[id], /opportunities, /insights, /properties, /team, /settings, más landing comercial en /.

2) Stack completo y arquitectura

  • Frontend: Next.js App Router, React 19, Tailwind v4, componentes reutilizables.
  • Backend app: Route Handlers en app/api/** + Server Actions para operaciones de leads.
  • Autenticación: better-auth con sesión en DB y requireSessionContext().
  • Multitenancy: scoping por agencyId derivado de sesión (no por query libre).
  • Persistencia: PostgreSQL + Prisma (schema robusto de operación comercial e inteligencia).
  • Integración de canales: Webhook Meta en /api/webhooks/meta + salida por Graph API.
  • Testing: Vitest (unit/integration) + Playwright E2E.

5) Cómo funciona la parte de IA hoy

Sí hay sistema de inteligencia funcionando. Hoy es un motor determinístico de reglas (no un modelo generativo LLM en producción).

  1. Ingreso de lead/mensajes por API o webhook Meta.
  2. Extracción de señales (presupuesto, zonas, timeline, tipo propiedad, objeciones, etc.).
  3. Scoring ponderado (0..100) + cálculo de urgencia/seriedad/prioridad.
  4. Recomendación de next best action según modo de política (conservador/agresivo).
  5. Reporte de confianza y detección de casos para revisión humana.
  6. Persistencia de outputs en Lead, LeadProfile, AiRun, AnalyticsEvent.
  7. Recalificación automática cuando entran nuevos mensajes al hilo.

Lógica de calificación (resumen práctico)

  • Señales fuertes: presupuesto claro, zona definida, timeline, tipo de propiedad, profundidad conversacional.
  • Urgencia: timeline corto y lenguaje de prioridad elevan score operativo.
  • Seriedad: cantidad/calidad de señales + consistencia en mensajes.
  • Objeciones penalizan probabilidad de cierre.
  • Política de agencia ajusta umbral de urgencia y estrategia de recomendación.

6) Qué tenemos vs qué falta

Ya tenemos (real hoy)

  • Auth + sesión + scoping tenant en rutas críticas.
  • CRUD operativo de acciones clave sobre leads.
  • Persistencia completa en PostgreSQL.
  • Integración Meta inbound/outbound funcional para MVP.
  • Suite de tests unit, integration y E2E en camino crítico.

Falta / simplificado

  • Roles/permisos avanzados (RBAC fino).
  • Colas/reintentos robustos para jobs críticos.
  • IA generativa productiva (explicabilidad + guardrails + evals online).
  • Observabilidad empresarial (SLO/alertas/tracing).
  • Backoffice de auditoría y billing/planes multi-tenant.

7) Flujo completo para probar como usuario real

Paso A — Preparación técnica

  1. Instalar dependencias: npm install
  2. Configurar .env desde .env.example (DB + Better Auth + Meta opcional).
  3. Migrar base: npm run db:migrate
  4. Crear usuario operador: npm run create-account -- --new-agency --email tu@email --password 'TuPass123!' --name 'Operador'
  5. Levantar app: npm run dev

Paso B — Flujo funcional mínimo del negocio

  1. Ingresar en /sign-in con el operador creado.
  2. Ir a /settings y registrar conexión WhatsApp/Instagram (si aplica).
  3. Crear lead por intake (webhook Meta o endpoint intake).
  4. Verificar que aparece en /leads.
  5. Abrir detalle del lead y ejecutar operaciones: etapa, owner, tarea, nota, visita.
  6. Ir a /team y confirmar impacto operativo.
  7. Ir a /insights y validar señales estratégicas derivadas.

Paso C — Prueba de mensajería Meta (si está conectado)

  1. Desde /settings usar “Prueba de envío”.
  2. Desde /leads, crear borrador y aprobar envío.
  3. Simular/recibir webhook de estado y validar actualización de delivery.

8) Testing automatizado actual

  • Unit: pipeline de calificación, seguridad Meta, parser payloads, utilidades y rutas de mensajes.
  • Integration: flujo lead intake→operación y mensajería Meta workflow.
  • E2E: journey principal de lead + aprobación/envío de mensaje Meta.

Comandos: npm test npm run test:e2e

9) Evaluación honesta: ¿“tenemos un sistema de IA”?

Sí, en el sentido operativo del MVP: hay un motor de inteligencia que extrae, puntúa, prioriza y recomienda acciones con persistencia y trazabilidad.

No, si la definición exige un stack avanzado de modelos fundacionales en línea con aprendizaje continuo. Hoy el núcleo es rule-based, sólido para validar negocio, pero todavía no es una “plataforma de IA avanzada” en versión final.

10) Prioridades recomendadas de mejora

  1. Confiabilidad de mensajería: colas + retries + dead-letter + observabilidad.
  2. Calidad de insights: explicaciones causales por lead y por etapa.
  3. Métricas de negocio cerradas: visitas calificadas, conversión por owner, tiempo a primera acción.
  4. Permisos por rol y controles anti-fuga de datos entre agencias.
  5. Evolución IA: capa LLM opcional sobre el motor determinístico con evals y control de costo.
